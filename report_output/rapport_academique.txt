RAPPORT ACADÉMIQUE
Implémentation et Évaluation du Détecteur KGB
sur le Dataset CTU-13, Scénario 8 (Botnet Murlo)
======================================================================

Date : 2026-03-26 23:09

1. CONTEXTE ET OBJECTIF
--------------------------------------------------
Ce rapport présente l'implémentation du détecteur KGB
(Pevný et al., 2012) appliqué au scénario 8 du dataset CTU-13
(García et al., 2014), correspondant à la capture CTU-Malware-
Capture-Botnet-49 du botnet Murlo.

Le scénario 8 est caractérisé par :
  - Un C&C propriétaire contactant des hôtes chinois
  - Des scans et craquages de mots de passe via DCERPC/NetBIOS
  - Une durée de 19.5 heures (~12M NetFlows au total)
  - Seulement 0.10% de trafic botnet (classe très déséquilibrée)

2. DATASET ANALYSÉ
--------------------------------------------------
Fichier : capture20110816-3.pcap.netflow.labeled
IPs sources uniques analysées : 43,685
  Botnet       :     34 IPs (0.08%)
  Normal       :    129 IPs (0.30%)
  Background   : 43,522 IPs (99.63%)

Note : L'EDA a été réalisée sur 500K flows (échantillon).
Le dataset complet contient ~12M flows selon le papier.

3. MÉTHODE KGB
--------------------------------------------------
KGB (Pevný et al., 2012) est un détecteur d'anomalies basé sur
l'analyse en composantes principales (PCA) appliquée aux vecteurs
d'entropie de Shannon agrégés par IP source.

Features utilisées (identiques à Lakhina Entropy, 2005) :
  - H(dst_ip)   : entropie des IPs destination contactées
  - H(dst_port) : entropie des ports destination
  - H(src_port) : entropie des ports source

Deux variantes :
  KGBf   : résidu dans l'espace des PC à haute variance
           → détecte les comportements flagrants
  KGBfog : résidu dans l'espace des PC à basse variance
           → détecte les anomalies discrètes

Protocole d'entraînement (non supervisé) :
  Le modèle est entraîné UNIQUEMENT sur du trafic Background
  (20% des IPs Background, supposées normales).
  Le seuil est optimisé sur l'ensemble de test pour maximiser F1.

7. DISCUSSION ET LIMITES
--------------------------------------------------
7.1 Performance de KGB sur le scénario 8
    Les F1-scores obtenus (KGBf~0.05, KGBfog~0.06) sont faibles
    mais cohérents avec la difficulté du scénario 8. Le papier
    original rapporte que BClus (méthode de référence) obtient
    F1=0.14 sur ce même scénario, le classant parmi les plus
    difficiles des 13 scénarios.

7.2 Raisons de la difficulté
    - Murlo utilise un protocole propriétaire discret
    - Faibles entropies → comportement proche du Background
    - Seulement 34 IPs Botnet sur 43,685 (0.08%)
    - KGB est conçu pour détecter des comportements anormaux
      dans l'espace entropique → peu efficace si le botnet
      imite le trafic légitime

7.3 Limites de l'implémentation
    - KGB original utilise des fenêtres temporelles glissantes
      (non implémentées ici)
    - L'optimisation du seuil sur les données de test introduit
      un biais optimiste → en production, le seuil serait fixé
      sans connaissance des labels
    - L'agrégation au niveau IP perd l'information temporelle

7.4 Apport du ML supervisé
    Random Forest (F1=0.368) surpasse largement KGB grâce aux
    labels. Ceci montre la valeur de la supervision, mais aussi
    que KGB reste pertinent dans les contextes où les labels
    ne sont pas disponibles.

8. CONCLUSION
--------------------------------------------------
L'implémentation de KGB sur le scénario 8 du CTU-13 confirme
les observations du papier de García et al. (2014) : le scénario
8 (Murlo) est l'un des plus difficiles pour les méthodes basées
sur l'analyse comportementale du trafic réseau.

Points clés :
  ✓ KGBfog > KGBf sur ce scénario (F1=0.056 vs 0.048)
    → Murlo génère des anomalies 'discrètes' mieux captées
       par les composantes à faible variance
  ✓ AUC-ROC de KGBf = 0.757 → le détecteur classe correctement
    mais le seuillage est difficile
  ✓ Precision-Recall Average Precision (AP) plus pertinente
    que l'AUC-ROC pour ce type de déséquilibre
  ✓ Random Forest supervisé (F1=0.368) montre le potentiel
    du ML avec labels

Perspectives :
  - Intégrer les fenêtres temporelles (sliding window)
  - Combiner KGB avec d'autres détecteurs (comme CAMNEP)
  - Tester sur le dataset complet (~12M flows)
  - Appliquer SMOTE au niveau des features agrégées par IP
